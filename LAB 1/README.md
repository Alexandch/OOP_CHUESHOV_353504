# Paint App на Python с Tkinter

## 1. Описание проекта

**Paint App** — это простое графическое приложение, разработанное на языке Python с использованием библиотеки Tkinter. Проект позволяет пользователям рисовать базовые фигуры, такие как круги, прямоугольники и линии, на холсте, а также предоставляет дополнительные функции: отмену и повтор действий, сохранение и загрузку состояния холста, изменение цвета фона. Приложение построено с использованием принципов объектно-ориентированного программирования (ООП), включая наследование, инкапсуляцию и полиморфизм, и реализует шаблон проектирования "Команда" для управления историей действий. Оно демонстрирует навыки программирования графического интерфейса и проектирования программного обеспечения.

### Краткое описание классов:
- **Shape**: Абстрактный базовый класс, определяющий общие атрибуты и методы для всех фигур.
- **Circle, Rectangle, Line**: Конкретные классы фигур, наследующиеся от `Shape`, представляющие определённые объекты для рисования.
- **CanvasApp**: Управляет холстом и фигурами, нарисованными на нём.
- **Command**: Абстрактный базовый класс для команд, поддерживающий функции отмены и повтора.
- **AddShapeCommand, EraseShapeCommand, MoveShapeCommand, SetBackgroundCommand**: Конкретные классы команд для добавления, удаления, перемещения фигур и установки цвета фона.
- **History**: Управляет историей действий для отмены и повтора.
- **PaintApp**: Основной класс приложения, объединяющий все компоненты.

## 4. Функциональные требования

Приложение Paint поддерживает следующие функции, каждая из которых описана подробно:

### 4.1. Рисование фигур
- **Круг**: Пользователь может нарисовать круг, указав координаты центра (x, y), радиус и цвет.
- **Прямоугольник**: Пользователь может нарисовать прямоугольник, указав координаты верхнего левого угла (x, y), ширину, высоту и цвет.
- **Линия**: Пользователь может нарисовать линию, указав начальную точку (x1, y1), конечную точку (x2, y2) и цвет.

### 4.2. Удаление фигур
- Пользователь может удалить любую фигуру, указав её уникальный идентификатор (ID), что приводит к её исчезновению с холста.

### 4.3. Перемещение фигур
- Пользователь может переместить фигуру, указав её ID и новые координаты (x, y), изменяя её положение на холсте.

### 4.4. Изменение цвета фона
- Пользователь может изменить цвет фона холста, выбрав новый цвет.

### 4.5. Сохранение и загрузка холста
- **Сохранение**: Пользователь может сохранить текущее состояние холста (цвет фона и все фигуры) в файл формата JSON.
- **Загрузка**: Пользователь может загрузить ранее сохранённое состояние холста из файла JSON, восстанавливая все фигуры и цвет фона.

### 4.6. Отмена и повтор действий
- **Отмена (Undo)**: Пользователь может отменить последнее действие (например, добавление, удаление или перемещение фигуры).
- **Повтор (Redo)**: Пользователь может повторить ранее отменённое действие.

### 4.7. Прокрутка холста
- Холст поддерживает горизонтальную и вертикальную прокрутку, что позволяет видеть фигуры, выходящие за пределы видимой области.

## 5. Устойчивость к пользовательскому вводу

Приложение разработано с учётом устойчивости ко всем типам пользовательского ввода:
- Если пользователь пытается удалить или переместить несуществующую фигуру (указав неверный ID), отображается сообщение об ошибке.
- Все команды проверяются на наличие достаточного количества параметров; при отсутствии или некорректности ввода выдаются соответствующие сообщения об ошибке.
- Приложение не завершает работу аварийно при любых действиях пользователя, обеспечивая стабильный опыт использования.

## 6. Набор тестовых случаев

Следующие тестовые случаи проверяют функциональность приложения:

### Тест 1: Добавление фигур
- **Действие**: Добавить круг, прямоугольник и линию с определёнными параметрами.
- **Ожидаемый результат**: Все фигуры появляются на холсте с заданными атрибутами.

### Тест 2: Удаление фигуры
- **Действие**: Удалить фигуру по её ID.
- **Ожидаемый результат**: Указанная фигура исчезает с холста.

### Тест 3: Перемещение фигуры
- **Действие**: Переместить фигуру на новые координаты, указав её ID.
- **Ожидаемый результат**: Фигура перемещается в новое положение.

### Тест 4: Изменение цвета фона
- **Действие**: Установить новый цвет фона.
- **Ожидаемый результат**: Фон холста обновляется до выбранного цвета.

### Тест 5: Сохранение и загрузка
- **Действие**: Сохранить холст в файл, затем загрузить его обратно.
- **Ожидаемый результат**: Загруженный холст соответствует сохранённому состоянию, включая фигуры и фон.

### Тест 6: Отмена и повтор
- **Действие**: Добавить фигуру, отменить действие, затем повторить его.
- **Ожидаемый результат**: Фигура исчезает при отмене и появляется снова при повторе.

### Тест 7: Прокрутка холста
- **Действие**: Добавить фигуры за пределами видимой области.
- **Ожидаемый результат**: Пользователь может прокрутить холст, чтобы увидеть все фигуры.

## 7. Удобный интерфейс управления

Приложение оснащено интуитивно понятным графическим интерфейсом пользователя (GUI) для удобства использования:
- **Меню "Файл"**: Содержит опции для сохранения, загрузки и выхода из приложения.
- **Меню "Правка"**: Включает команды для отмены/повтора действий, добавления фигур, удаления, перемещения фигур и установки цвета фона.
- **Кнопки "Отмена" и "Повтор"**: Быстрый доступ к функциям undo и redo.
- **Диалоговые окна**: Используются для ввода параметров фигур, упрощая взаимодействие с пользователем.
